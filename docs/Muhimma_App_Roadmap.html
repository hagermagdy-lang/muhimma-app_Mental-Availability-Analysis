Features Proposed

Muhimma is a web-based Mental Availability analytics dashboard built on the Ehrenberg-Bass Institute framework. It enables brand researchers and marketers to upload raw survey data (Wide Text CSV) and instantly visualize key brand health metrics: Mental Penetration (MPen), Network Size (NS), and Mental Market Share (MMS).

Tech Stack: React 19 · Vite 8 · TailwindCSS 3.4 · React Router v7 · PapaParse · Recharts 3.8

Section 05

Open Questions for Stakeholders

① Target Audience

Is the primary user a professional researcher who needs raw data tables and a fast workflow, or a Brand Manager who needs high-level executive visuals and export-ready reports? This determines whether we prioritize Phase 3 (Export) or Phase 2 (Visualizations) first.

② Brand Normalization Approach

Should we use automatic fuzzy matching (e.g., "Pepsy" auto-corrects to "Pepsi") or a manual merge UI where the user decides? Automatic matching risks false merges in niche or regional brand datasets.

③ Data Format Strategy

Should we enforce the Wide Text CSV format exclusively (simpler, faster), or invest in supporting SPSS exports and Long-Format CSV? Supporting more formats broadens the audience but increases parsing complexity significantly.

④ Deployment Architecture

Is this intended to remain a client-side-only tool (no backend, CSV never leaves the browser), or should we plan for a backend server to enable shareable links, saved projects, and user accounts? This is a foundational decision that affects Phase 3 and 4.
